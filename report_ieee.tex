%  ABSTRACT

\begin{abstract}
Deadlock detection and prevention remain among the most critical challenges in
operating systems design. Classical approaches such as the Banker's Algorithm and
Wait-For Graph (WFG) analysis are reactive—they can confirm a deadlock only after
it has fully materialised, and they require complete a priori knowledge of resource
needs. In this paper we present \textbf{DeadlockGNN}, an end-to-end AI framework
that reformulates deadlock prediction as a graph classification problem. We model
each OS state as a typed Resource Allocation Graph (RAG) and apply a
Relational Graph Convolutional Network (RGCN) to perform binary deadlock
classification on individual snapshots. For temporal forecasting we extend the
model with a Gated Recurrent Unit (GRU) that processes sequences of eight
consecutive RAG snapshots and predicts whether the \textit{next} OS tick will
result in a deadlock. The static model is trained on a synthetic corpus of
200,000~graphs; its learned spatial weights are transferred into the temporal
encoder via cross-domain transfer learning. A live integration with the
xv6-riscv kernel running inside QEMU closes the loop between trained AI and a
real running operating system. Our temporal model achieves \textbf{93.74\%}
accuracy, \textbf{94.38\%} F1-score, and \textbf{0.9831} AUC-ROC at a
per-sequence inference latency of only \textbf{2.9\,ms}, demonstrating
that GNN-based proactive deadlock prediction is both feasible and
computationally practical for real OS workloads.
\end{abstract}

deadlock prediction, graph neural network, relational GCN, temporal forecasting,

%  I. INTRODUCTION
% ─────────────────────────────────────────────
\section{Introduction}

A deadlock arises when a set of processes each holds at least one resource while
waiting for a resource held by another process in the set, forming a circular
dependency from which none can escape without external intervention
\cite{silberschatz2018}. Despite decades of research, deadlocks remain a
practical concern in concurrent software including databases, distributed systems,
and multi-threaded applications \cite{lu2008learning}.

Traditional approaches fall into four categories: \textit{prevention} (restrict
the order in which resources may be requested), \textit{avoidance} (Banker's
Algorithm \cite{dijkstra1965}), \textit{detection} (cycle detection on the
Wait-For Graph \cite{coffman1971}), and \textit{recovery} (process termination
or resource pre-emption). All four either impose significant runtime overhead,
require full knowledge of future resource needs, or act only after the system
has already deadlocked.

Machine learning offers a complementary paradigm: instead of reasoning about
resource states symbolically, learn the \textit{structural signature} of
near-deadlock configurations from data. Prior work by Peng
et al.~\cite{peng2020} applied LSTM models to kernel trace sequences, while
Shi et al.~\cite{shi2021} used GNNs on static program dependency graphs. Neither
produced a system capable of processing live OS graphs in real time.

The key contributions of this paper are:
\begin{enumerate}
  \item A typed 7-dimensional node feature encoding for Resource Allocation Graphs
        that captures process role, contention pressure, and resource utilisation.
  \item A 3-layer \textbf{Relational GCN} trained on 200,000 synthetic RAG snapshots
        for instantaneous ($<$1\,ms) static deadlock classification.
  \item A \textbf{Temporal RGCN-GRU} hybrid that forecasts deadlock at the next
        OS tick from a sequence of 8 historical snapshots, using transfer learning
        from the static model.
  \item Live instrumentation of the \textbf{xv6-riscv} kernel and a real-time
        inference pipeline bridging kernel events to the GNN dashboard.
  \item Monte-Carlo \textbf{Shapley attribution} for explainability — identifying
        which processes and resources cause the predicted risk.
\end{enumerate}

%  II. LITERATURE SURVEY
% ─────────────────────────────────────────────
\section{Literature Survey}

\subsection{Classical Deadlock Theory}
Coffman et al.~\cite{coffman1971} established the four necessary conditions for
deadlock: mutual exclusion, hold-and-wait, no pre-emption, and circular wait.
Their Wait-For Graph formulation remains the theoretical foundation of all
detection-based approaches and directly motivates our graph-based ML formulation.

Dijkstra's Banker's Algorithm \cite{dijkstra1965} provides a polynomial-time
safe-state check but requires complete knowledge of maximum resource claims — a
requirement rarely satisfied in practice.

\subsection{Graph Neural Networks for System Analysis}
Kipf and Welling~\cite{kipf2017} introduced semi-supervised classification with
Graph Convolutional Networks (GCNs), establishing the message-passing paradigm
that our work builds upon. Their formulation, however, treats all edges
identically.

Schlichtkrull et al.~\cite{schlichtkrull2018} extended GCNs to heterogeneous
graphs via Relational GCNs (R-GCNs), using distinct weight matrices per edge
type. This is critical in our RAG setting where ``request'' and ``assignment''
edges carry opposite semantic meaning.

Hamilton et al.~\cite{hamilton2017} proposed GraphSAGE for inductive learning on
unseen graphs, which we evaluated as a baseline against R-GCN.

\subsection{Temporal Graph Learning}
Seo et al.~\cite{seo2018} combined GCNs with GRUs for spatio-temporal graph
forecasting. Their architecture, designed for traffic prediction, directly
inspired our RGCN-GRU hybrid for OS state evolution.

Pareja et al.~\cite{pareja2020} proposed EvolveGCN, which adapts GCN weights
over time using an RNN. Unlike EvolveGCN, our approach keeps the spatial encoder
fixed (transferred from a stronger static model) and solely trains the GRU to
capture temporal dynamics.

\subsection{ML-Based Deadlock and Concurrency Bug Detection}
Lu et al.~\cite{lu2008learning} performed the seminal empirical study of
concurrency bugs in real-world software, finding that $\sim$30\% involved
deadlocks, motivating automated detection.

Peng et al.~\cite{peng2020} applied sequence models (LSTM) to kernel call trace
features for deadlock prediction, achieving 89\% accuracy but without graph
structure. Our work shows that incorporating RAG topology improves this to
over 93\%.

Shi et al.~\cite{shi2021} used GNNs on program dependence graphs for general
concurrency bug detection. Their work operates at the static code level, whereas
we target dynamic OS runtime states.

\subsection{Explainability in GNNs}
Lundberg and Lee~\cite{lundberg2017} introduced SHAP (SHapley Additive
exPlanations), a unified framework for feature attribution. We apply a
Monte-Carlo variant of Shapley values directly to graph nodes, identifying
which processes and resources are most responsible for a predicted deadlock.

Ying et al.~\cite{ying2019} proposed GNNExplainer for subgraph-level
explanations in GNNs. Our node-level Shapley approach provides more interpretable
output for the OS domain where individual processes and resources have clear
semantic identity.

\section{Proposed Methodology}

\subsection{Problem Formulation}

Let $G_t = (V_t, E_t)$ denote the Resource Allocation Graph at OS tick $t$,
where $V_t = \mathcal{P}_t \cup \mathcal{R}_t$ is the union of active process and
resource nodes, and $E_t$ consists of directed edges of two types:
$\tau_e \in \{\textsc{req}, \textsc{asgn}\}$.

\textbf{Static task}: Given $G_t$, predict $y_t \in \{0,1\}$ (deadlocked or safe).

\textbf{Temporal task}: Given the sequence
$\mathcal{S}_t = (G_{t-7}, G_{t-6}, \ldots, G_t)$, predict $y_{t+1}$.

\subsection{Wait-For Graph \& Ground Truth}

\begin{algorithm}[H]
\caption{Deadlock Detection via Wait-For Graph}
\begin{algorithmic}[1]
\REQUIRE RAG $G = (V, E)$
\ENSURE deadlocked $\in \{\textit{true},\textit{false}\}$, cycle path
\STATE $\text{WFG} \leftarrow \emptyset$
\FOR{each resource $r \in \mathcal{R}$}
  \STATE $H \leftarrow \{p : (r \to p) \in E\}$ \COMMENT{holders}
  \STATE $W \leftarrow \{p : (p \to r) \in E\}$ \COMMENT{waiters}
  \FOR{$w \in W$, $h \in H$, $w \neq h$}
    \STATE $\text{WFG} \leftarrow \text{WFG} \cup \{(w \to h)\}$
  \ENDFOR
\ENDFOR
\STATE Run DFS on WFG; return (\textit{true}, cycle) if back-edge found
\end{algorithmic}
\end{algorithm}

\subsection{Node Feature Encoding}

Each node $v$ is encoded as $\mathbf{f}_v \in \mathbb{R}^7$:

\begin{equation}
\mathbf{f}_v = \bigl[
  \mathbb{1}_P,\;
  \mathbb{1}_R,\;
  \tfrac{\deg(v)}{\max \deg},\;
  \tfrac{d^{in}(v)}{d^{out}(v)+1},\;
  |\mathcal{E}^{\text{req}}_v|,\;
  |\mathcal{E}^{\text{asgn}}_v|,\;
  \tfrac{|\text{holders}(v)|}{\text{cap}(v)}
\bigr]
\end{equation}

where $\mathbb{1}_P, \mathbb{1}_R$ are node-type indicators, the degree and ratio
terms capture structural centrality, and the final term encodes resource
utilisation (zero for process nodes).

\subsection{Static RGCN Model}

The RGCN propagation rule for relation $r$ is:

\begin{equation}
\mathbf{h}_i^{(l+1)} = \sigma\!\left(
  \sum_{r \in \mathcal{R}} \sum_{j \in \mathcal{N}_r(i)}
  \frac{1}{|\mathcal{N}_r(i)|} \mathbf{W}_r^{(l)} \mathbf{h}_j^{(l)}
\right)
\end{equation}

Each relation type $r \in \{\textsc{req}, \textsc{asgn}\}$ maintains its own
weight matrix $\mathbf{W}_r$, enabling the model to learn that ``waiting for''
and ``holding'' carry fundamentally different structural roles in deadlock
formation.

\begin{algorithm}[H]
\caption{RGCN Static Forward Pass}
\begin{algorithmic}[1]
\REQUIRE $\mathbf{X} \in \mathbb{R}^{n \times 7}$, edge list $E$, types $T$, batch $b$
\STATE $\mathbf{H} \leftarrow \text{ReLU}(\text{RGCNConv}_1(\mathbf{X}, E, T))$
\STATE $\mathbf{H} \leftarrow \text{ReLU}(\text{RGCNConv}_2(\mathbf{H}, E, T))$
\STATE $\mathbf{H} \leftarrow \text{RGCNConv}_3(\mathbf{H}, E, T)$
\STATE $\mathbf{z} \leftarrow \text{GlobalAddPool}(\mathbf{H}, b)$ \COMMENT{$\in \mathbb{R}^{64}$}
\STATE $\hat{y} \leftarrow \sigma\!\left(\text{MLP}(\mathbf{z})\right)$
\RETURN $\hat{y}$
\end{algorithmic}
\end{algorithm}

\subsection{Temporal RGCN-GRU Model}

\begin{algorithm}[H]
\caption{Temporal Forward Pass with Transfer Learning}
\begin{algorithmic}[1]
\REQUIRE Sequence $\mathcal{S} = [G_0, \ldots, G_7]$
\STATE \textbf{Transfer:} load $\{$conv$_1$, conv$_2$, conv$_3\}$ from static checkpoint
\FOR{$t = 0$ \TO $7$}
  \STATE $\mathbf{e}_t \leftarrow \text{RGCN\_Encoder}(G_t)$ \COMMENT{shared weights}
\ENDFOR
\STATE $\mathbf{Z} \leftarrow [\mathbf{e}_0; \ldots; \mathbf{e}_7] \in \mathbb{R}^{8 \times 64}$
\STATE $\_, \mathbf{h}_T \leftarrow \text{GRU}(\mathbf{Z},\; \text{layers}=2,\; \text{hidden}=128)$
\STATE $\hat{y}_{t+1} \leftarrow \sigma\!\left(\text{MLP}(\mathbf{h}_T[-1])\right)$
\RETURN $\hat{y}_{t+1}$
\end{algorithmic}
\end{algorithm}

The RGCN encoder weights are frozen at the values learned from 200,000 static
graphs, acting as a powerful pre-trained feature extractor. Only the GRU and
classifier head train on the temporal dataset of 50,000 sequences.

\subsection{Live Kernel Integration}

We instrument \texttt{spinlock.c} and \texttt{proc.c} in the xv6-riscv kernel
to emit structured trace events:

\begin{verbatim}
[GNN_TRACE] LOCK_ACQUIRE P5 virtio_disk 1642
[GNN_TRACE] PROCESS_SLEEP P3 proc 1645
\end{verbatim}

A Python listener reads these from a UNIX FIFO pipe and incrementally updates
the live RAG in $O(1)$ per event (edge addition/removal only). The GNN
runs inference on the updated RAG every second.

\subsection{Shapley Attribution (XAI)}

\begin{algorithm}[H]
\caption{Monte-Carlo Shapley Node Attribution}
\begin{algorithmic}[1]
\REQUIRE Model $f$, graph $G$ with $n$ nodes, samples $T$
\STATE $\boldsymbol{\phi} \leftarrow \mathbf{0}^n$
\FOR{$t = 1$ \TO $T$}
  \STATE $S \leftarrow$ random subset of $\{1,\ldots,n\}$
  \FOR{each $i \notin S$}
    \STATE $\boldsymbol{\phi}[i] \mathrel{+}= f(G|_{S \cup \{i\}}) - f(G|_S)$
  \ENDFOR
\ENDFOR
\RETURN $\boldsymbol{\phi} / T$
\end{algorithmic}
\end{algorithm}

The high precision (96.30\%) indicates very few false alarms — a critical property

costly.

\subsection{Key Observations}

\begin{enumerate}
  \item \textbf{RGCN outperforms GraphSAGE}: The relational edge formulation
        improves F1 by $\sim$4\% over a plain GCN baseline, confirming that
        edge type semantics are essential for deadlock topology learning.

  \item \textbf{Transfer learning is critical}: Initialising the temporal encoder
        with static weights reduces convergence from $>$100 epochs to 30 epochs
        while improving final F1 by 3.8\%.

  \item \textbf{Sequence length matters}: Ablation over sequence lengths
        $T \in \{2, 4, 6, 8\}$ shows F1 improving monotonically, with diminishing
        returns beyond $T=8$.

  \item \textbf{Sub-3\,ms latency}: The 2.9\,ms per-sequence inference time is well
        within the $\sim$10\,ms OS scheduling quantum, making true real-time
        monitoring feasible.
\end{enumerate}

\section{Conclusion}

We presented DeadlockGNN, a system that reframes OS deadlock prediction as a
typed graph classification and temporal forecasting problem. By modelling OS
state as a heterogeneous Resource Allocation Graph and applying Relational GCN
with attention to edge semantics, we achieve 91–93\% accuracy on static snapshots.
The temporal extension, which processes sequences of eight consecutive snapshots
through a shared RGCN encoder followed by a two-layer GRU, achieves 93.74\%
accuracy and 0.9831 AUC-ROC — substantially outperforming prior LSTM-based
approaches that ignore graph structure.

Cross-domain transfer learning, injecting spatial weights from a 200k-graph static
model into the temporal encoder, proves essential for rapid convergence and
superior generalisation. Live integration with xv6-riscv demonstrates that the
system can monitor a real kernel at sub-3\,ms latency per inference.

\textbf{Future scope} includes: (1)~extending the kernel integration to
production kernels (Linux via eBPF probes); (2)~online/continual learning
where the model adapts to new workload patterns without full retraining;
(3)~graph-level attention mechanisms for interpretable deadlock path highlighting;
(4)~multi-GPU workload prediction using heterogeneous resource graphs.
